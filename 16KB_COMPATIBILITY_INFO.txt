16KB PAGE SIZE SUPPORT - TECHNICAL DETAILS
================================================================================

WHAT IS 16KB PAGE SIZE?
- Traditional Android devices use 4KB memory page sizes
- Android 15+ supports devices with 16KB memory pages (e.g., Pixel 9)
- Devices with 16KB pages provide better memory efficiency and performance
- Apps must be compatible with BOTH 4KB and 16KB page layouts

COMPATIBILITY FEATURES ENABLED:

1. NATIVE LIBRARY ALIGNMENT
   ✅ RunAnywhere SDK native libraries are built with proper alignment
   ✅ Supports both 4KB (arm64-v8a, armeabi-v7a, x86_64) and 16KB page sizes
   ✅ Android NDK toolchain automatically handles alignment

2. BUILD CONFIGURATION ENABLED
   ✅ android.support16KPageSize = true (in gradle.properties)
   ✅ ABI filters configured for multiple architectures
   ✅ Bundle configuration optimized for 16KB-compatible devices

3. LIBRARIES INCLUDED
   Native Libraries (16KB-compatible):
   - libonnxruntime.so (ONNX runtime for voice models)
   - librac_backend_llamacpp.so (LLaMA C++ inference)
   - librac_backend_onnx.so (ONNX backend)
   - libsherpa-onnx-*.so (Speech-to-Text models)
   - librunanywhere_jni.so (RunAnywhere SDK JNI)
   - libvad_jni.so (Voice Activity Detection)
   
   All libraries are pre-built with 16KB page alignment support by RunAnywhere.
